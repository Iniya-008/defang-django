Time Complexity Basics

Time complexity estimates how running time grows with input size n.

  • O(1): constant
  • O(log n): logarithmic
  • O(n): linear
  • O(n log n): e.g., mergesort
  • O(n^2): nested loops

Focus on the dominant term; drop constants.

Arrays in CSE

Arrays store items contiguously. Access by index is O(1).

Insert/delete at end can be O(1) amortized; middle is O(n).

Sorting Overview

Bubble (O(n^2)), Merge (O(n log n)), Quick (avg O(n log n)).

Stable keeps equal items ordered; in-place uses little extra memory.
